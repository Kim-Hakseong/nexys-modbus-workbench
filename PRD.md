# PRD.md — Nexys Modbus Workbench

## 1. 목적

Modbus Poll(마스터 폴링 도구)과 ModScan의 핵심 기능을 대체하는 사내 표준 Modbus 진단 도구.
현장 엔지니어가 PLC/계측기/보호계전기 등 Modbus 슬레이브 장비를 **연결 → 폴링 → 값 확인 → 쓰기 → 트래픽 분석**하는 전체 워크플로우를 하나의 exe로 처리한다.

## 2. 사용자와 시나리오

- 사용자: 넥시스 시스템사업부 엔지니어 (Windows 노트북, 현장/사무실)
- 시나리오 A: TCP로 장비에 붙어 Holding Register를 500ms 주기로 폴링하면서 float 값으로 모니터링
- 시나리오 B: USB-RS485 컨버터로 RTU 장비에 붙어 코일 상태 확인 후 FC05로 코일 토글
- 시나리오 C: 통신이 안 될 때 raw TX/RX hex 로그를 보고 CRC/타임아웃/exception 원인 분석
- 시나리오 D: 프로젝트별 접속정보+폴 정의를 워크스페이스 파일로 저장, 다음 현장 방문 시 즉시 재사용

## 3. 기능 요구사항

### P0 — 이것 없으면 제품이 아님 (Modbus Poll 핵심 동등 기능)

| ID | 기능 | 수용 기준 |
|---|---|---|
| F-01 | Modbus TCP 마스터 연결 | IP/Port/UnitId 지정, 연결/해제, 끊김 시 자동 재접속(설정 가능) |
| F-02 | Modbus RTU 마스터 연결 (시리얼) | COM포트/보레이트/패리티/데이터·정지비트 설정, 요청 직렬화(한 번에 1요청) |
| F-03 | 읽기 FC01/02/03/04 | 슬레이브ID, 시작주소, 개수(코일≤2000, 레지스터≤125) 지정 폴링 |
| F-04 | 쓰기 FC05/06/15/16 | 쓰기 다이얼로그에서 값 입력 → 전송 → 응답/exception 표시 |
| F-05 | 주기 폴링 엔진 | Scan rate ms 단위 설정(최소 10ms), 폴별 독립 주기, 시작/정지 |
| F-06 | 데이터 포맷 표시 | Unsigned/Signed 16bit, Hex, Binary, 32bit Int/UInt/Float(워드오더 4종: AB CD, CD AB, BA DC, DC BA), 64bit Double, ASCII |
| F-07 | 통신 통계 | 폴별 Tx count / Valid Rx / Error count / 마지막 응답시간(ms) / 마지막 에러 종류 |
| F-08 | 에러 처리 | 타임아웃, CRC 오류, Modbus exception(01~0B) 코드별 명칭 표시 |
| F-09 | 트래픽 로그 | 타임스탬프 + 방향(TX/RX) + raw hex 프레임, 스크롤 뷰, 파일 저장(.txt) |
| F-10 | 주소 표기 토글 | 프로토콜 주소(0-base) ↔ PLC 주소(1-base, 40001 스타일) 전환 |
| F-11 | 워크스페이스 저장/로드 | 연결 설정 + 모든 폴 정의 + 별칭을 JSON 파일(.nmw)로 저장/복원 |
| F-12 | 단일 exe 배포 | win-x64 self-contained single-file, 더블클릭 실행, 설치 불요 |

### P1 — 있어야 Modbus Poll을 실제로 대체함

| ID | 기능 | 수용 기준 |
|---|---|---|
| F-13 | 멀티 폴 윈도우 | 탭 방식으로 여러 폴 정의를 동시에 폴링/표시 (동일 채널 공유) |
| F-14 | 레지스터 별칭(Alias) | 각 주소에 이름 부여, 그리드에 주소+별칭+값 표시 |
| F-15 | 셀 더블클릭 쓰기 | 폴 그리드에서 값 더블클릭 → 쓰기 다이얼로그 프리필 |
| F-16 | RTU over TCP | RTU 프레임을 TCP로 전송하는 게이트웨이 모드 |
| F-17 | 타임아웃/재시도 설정 | 응답 타임아웃 ms, 재시도 횟수, RTU 프레임간 지연 설정 |
| F-18 | 로그 필터 | 에러만 보기 토글, 로그 클리어, 최대 라인 수 제한(기본 5000) |

### P2 — 여유 시 (M10 이후 별도 사이클)

- Modbus ASCII 모드, 폴 데이터 CSV 내보내기, 값 변화 하이라이트, 다크/라이트 테마 토글, 간이 슬레이브 시뮬레이터 모드

## 4. 비기능 요구사항

| 항목 | 기준 |
|---|---|
| 안정성 | 24시간 연속 폴링 시 크래시/메모리 누수 없음 (로그 라인 수 제한, 링버퍼) |
| 성능 | 10ms scan rate에서 UI 프리즈 없음, 폴 10개 동시 동작 |
| 범용성 | Windows 10 1809+ / 11, .NET 런타임 설치 불요 |
| 파일 크기 | 단일 exe ≤ 90MB (self-contained 기준) |
| 언어 | UI 한국어 기본 (문자열은 상수 클래스로 분리해 향후 영어화 대비) |

## 5. 명시적 비범위 (하지 않는 것)

- Modbus 슬레이브(서버) 제품 기능 (테스트용 내부 슬레이브는 테스트 프로젝트에만 존재)
- 트렌드 차트/그래프, 스크립팅, OPC/MQTT 브리지
- 설치 프로그램(MSI), 자동 업데이트, 코드 서명

## 6. 마일스톤 (Ralph 루프 단위)

| M | 산출물 | DoD |
|---|---|---|
| M1 | 솔루션 스캐폴드 + `Nmw.Core` PDU 빌더/파서 + CRC16 | 골든 벡터 테스트 전체 통과 (DESIGN §7) |
| M2 | 프레이밍: MBAP(TCP), RTU 프레임 조립/타임아웃 기반 수신 | 프레이밍 단위 테스트 통과 |
| M3 | 트랜스포트: TcpTransport / SerialTransport / RtuOverTcp + 재시도·타임아웃 | 페이크 스트림 기반 테스트 통과 |
| M4 | 테스트 슬레이브(TCP) + 통합 테스트 (FC01~06,15,16 왕복) | 통합 테스트 통과, exception 케이스 포함 |
| M5 | 폴링 엔진 + 통계 + 데이터 포맷 변환기 | 포맷 변환 골든 벡터 + 폴링 스케줄 테스트 통과 |
| M6 | Avalonia UI 셸: 연결 다이얼로그, 폴 그리드 1개, 시작/정지 | 앱 실행, 테스트 슬레이브 대상 수동 스모크 시나리오 통과 |
| M7 | 쓰기 다이얼로그(FC05/06/15/16) + 주소 base 토글 + 별칭 | 쓰기 왕복 통합 테스트 + 스모크 통과 |
| M8 | 트래픽 로그 뷰 + 파일 저장 + 에러 카운터 표시 + 로그 제한 | 로그 링버퍼 테스트 + 스모크 통과 |
| M9 | 멀티 폴 탭 + 워크스페이스 저장/로드(.nmw) | 저장→로드 라운드트립 테스트 + 스모크 통과 |
| M10 | win-x64 single-file publish 검증 + README + 스모크 체크리스트 문서 | publish 명령 성공 확인 후 산출물 삭제, 문서 완비 |
